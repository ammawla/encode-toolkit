# Cite ENCODE: Publication-Ready Citations and Compliance

> Generate BibTeX, RIS, and acknowledgment text for ENCODE data -- ensuring
> your manuscript meets ENCODE data use policy before submission.

**Skill:** `cite-encode` | **Tools used:** `encode_track_experiment`, `encode_get_citations`, `encode_export_data`

---

## Scenario

You are preparing a manuscript using three ENCODE ATAC-seq experiments on human
pancreas. You need BibTeX for LaTeX, a data availability statement, and an
acknowledgment section -- all compliant with ENCODE data use policy.

## Step 1: Track Experiments to Auto-Link Publications

**You ask Claude:** "Track ENCSR133RZO so I can cite it properly."

**Claude calls:** `encode_track_experiment(accession="ENCSR133RZO", fetch_publications=True)`

```json
{"status": "tracked", "accession": "ENCSR133RZO", "publications_found": 1,
 "publications": [{"pmid": "32728249", "title": "Expanded encyclopaedias of DNA elements...",
   "journal": "Nature", "year": 2020, "doi": "10.1038/s41586-020-2493-4"}]}
```

When `fetch_publications=True` (the default), tracking automatically discovers and stores
associated publications. No manual linking required for papers in the ENCODE portal.

## Step 2: Export BibTeX for LaTeX

**You ask Claude:** "Export all my citations as BibTeX."

**Claude calls:** `encode_get_citations(export_format="bibtex")`

```bibtex
@article{Moore2020_ENCODE3,
  title   = {Expanded encyclopaedias of {DNA} elements in the human and mouse genomes},
  author  = {Moore, Jill E. and Purcaro, Michael J. and Pratt, Henry E. and others},
  journal = {Nature},
  volume  = {583},
  pages   = {699--710},
  year    = {2020},
  doi     = {10.1038/s41586-020-2493-4},
  note    = {Linked to ENCODE experiment ENCSR133RZO}
}
```

Paste directly into your `.bib` file. The `note` field ties each publication back to
its ENCODE accession -- useful when reviewers ask which paper corresponds to which dataset.

## Step 3: Export RIS for Reference Managers

**You ask Claude:** "I also need RIS format for Zotero."

**Claude calls:** `encode_get_citations(accession="ENCSR133RZO", export_format="ris")`

```ris
TY  - JOUR
TI  - Expanded encyclopaedias of DNA elements in the human and mouse genomes
AU  - Moore, Jill E.
AU  - Purcaro, Michael J.
JO  - Nature
VL  - 583
PY  - 2020
DO  - 10.1038/s41586-020-2493-4
ER  -
```

Import directly into Zotero, Endnote, or Mendeley via File > Import.

## Step 4: Generate Data Availability Statement

**You ask Claude:** "Write my data availability section for ENCSR133RZO, ENCSR388QZF, and ENCSR617GMR." Claude drafts from the ENCODE-compliant template:

> ATAC-seq data for human pancreas tissue were obtained from the ENCODE Project
> (https://www.encodeproject.org). Experiment accessions: ENCSR133RZO, ENCSR388QZF,
> ENCSR617GMR. All ENCODE data are freely available under unrestricted use policy.

Always list specific ENCSR accessions -- "data were obtained from ENCODE" alone is
insufficient for reproducibility.

## Step 5: Write the Acknowledgment Section

**You ask Claude:** "Draft an acknowledgment section for my ENCODE data."

> This work used data generated by the ENCODE Consortium (encodeproject.org). The
> ENCODE Project is funded by the National Human Genome Research Institute (NHGRI).
> We thank the Bradley Bernstein laboratory (Broad Institute) for generating the
> histone ChIP-seq data used in this study.

Lab-specific acknowledgments are drawn from tracked experiment metadata. NHGRI
acknowledgment is required by ENCODE data use policy.

## Step 6: Build a Supplementary Table

**You ask Claude:** "Export a supplementary table of all experiments I used."

**Claude calls:** `encode_export_data(format="tsv")` -- the output includes accession,
assay, target, biosample, organism, assembly, lab, and release date. Include this as
Supplementary Table S1.

## Best Practices

- **Cite both consortium and lab papers.** ENCODE Phase 3 (PMID 32728249) for the project broadly; individual experiment publications for specific datasets.
- **Check publication trust first.** Run `publication-trust` on each cited paper to catch retractions before submission.
- **Cite the published version, not the preprint.** Use `convert_article_ids` to check whether a bioRxiv preprint has a journal DOI.
- **Include ENCFF accessions in methods.** Beyond ENCSR accessions, list specific file accessions (peak files, signal tracks) you analyzed.
